# Documentation Intent

- **Production code**: write *how* — the code should express the mechanism clearly through naming, structure, and inline comments where the algorithm is non-obvious.
- **Test code**: write *what* — test names and assertions should describe the expected behavior and specification.
- **Commit messages**: write *why* — explain the motivation and reasoning behind the change, not a summary of what changed.
- **Code comments**: write *why not* — comments should explain rejected alternatives, constraints, and non-obvious trade-offs. Do not restate what the code already says.
