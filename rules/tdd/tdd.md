# TDD: Red-Green-Refactor

- Follow strict TDD red-green-refactor cycle.
- Write exactly one failing test first (red), then write the minimal implementation to make it pass (green), then refactor.
- Never implement more than one behavior at a time. Complete the full cycle before starting the next.
- Do not write production code without a failing test that demands it.
- Each cycle must be independently verifiable: test fails → test passes → code is clean.
