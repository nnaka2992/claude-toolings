# Git Rules

- Use conventional commit format: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`, `test:`, `ci:`, etc.
- Never use `git push --force`. Use `--force-with-lease` if a force push is absolutely necessary.
- Write concise commit messages focused on the "why" rather than the "what".
- Do not add Co-Authored-By lines to commit messages.
